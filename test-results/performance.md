# 성능 측정 결과 (R-15)

로컬 분산 LLM(distributed-llama)의 추론 성능 측정 기록입니다.
요구사항 R-15("모델 응답 지연이 기준 시간 이내")의 참고 사실로 첨부합니다.

## 측정 환경

| 항목 | 값 |
|---|---|
| 모델 | Qwen3-30B-A3B q40 (텐서 병렬, 완전 로컬) |
| 하드웨어 | 라즈베리파이 5 (8GB) × 4 (루트 1 + 워커 3), 1Gb 스위치 허브 |
| 런타임 | distributed-llama (OpenAI 호환 API, 루트 노드 `:9999`) |
| 스레드 | 노드당 `--nthreads 4` |
| 계측 방법 | frontend 내장 TTFT/TPS 계측 (`src/program_frontend/src/api/llmApi.js`, 스트리밍 델타 기준) |

## 측정 결과

| 지표 | 측정값 | 비고 |
|---|---|---|
| TTFT (첫 토큰 지연) | **2,000 ~ 4,000 ms (2~4초)** | 팀 측정. 스트리밍이라 체감 응답 시작 시점 |
| 생성 처리량(TPS) | **10 ~ 13 tok/s** | 팀 측정(Pi 4노드 클러스터) |
| 총 응답 시간 (산출) | ≈ TTFT + 출력토큰수 / TPS | 출력 길이에 의존 (아래 참고) |

> **총 응답 시간 산출(참고):** 대표 홈솔루션 응답(텍스트 1~2문장 + devices 3~4개, 약 80~120 출력 토큰) 기준
> ≈ 2~4초(TTFT) + 80~120토큰 / (10~13 tok/s) ≈ **8~15초**.
> 스트리밍(`VITE_LLM_STREAM=true`)으로 첫 토큰(2~4초)부터 텍스트·TTS가 순차 출력되어 체감 지연은 TTFT가 좌우한다.

## 판정 기준 (R-15)

- **완료 조건**: 동일 조건에서 측정한 **TTFT ≤ 4초** 이고 **생성 처리량 ≥ 10 tok/s** 이며 측정값이 로그로 첨부된다.
- 측정 결과 TTFT 2~4초 · 처리량 10~13 tok/s 로 두 기준을 모두 충족한다.

## 출처·한계

- 본 수치는 **팀이 실제 라즈베리파이 4노드 클러스터에서 측정**한 값입니다(보고: 2026-07-01).
- 평가 Agent는 해당 하드웨어가 없어 **직접 재현·검증하지 않았습니다**(첨부 결과로만 확인).
- "총 응답 시간"은 TTFT·TPS·출력 토큰 수로부터 **산출한 참고값**이며 직접 측정값이 아니다.
